Git - jest to system kontroli wersji, pozwalający na śledzenie zmian w kodzie oraz polepsza pracę zespołową
→Aby zacząć korzystać z GIT-a wystarczy pobrać Git bash. Można utworzyć projekt komendą lub otworzyć folder, w którym chcemy śledzić zmiany, pomocą Git Bash.
→Aby zaoszczędzić czas oraz miejsce w zdalnym repozytorium tworzymy plik o nazwie '.gitignore'. Posiada on w sobie rozszerzenia plików, których ma nie zapisywać przez co na zdalnym repozytorium jest to co niezbędne aby kontynuować pracę z każdego miejsca na świecie z dostępem do internetu.
→Git posiada trzy stany, w których mogą znajdować się pliki: zatwierdzony, zmodyfikowany oraz oczekujący w poczekalni. Zatwierdzony oznacza, że dane zostały bezpiecznie zachowane w Twojej lokalnej bazie danych. Zmodyfikowany oznacza, że plik został zmieniony, ale zmiany nie zostały wprowadzone do bazy danych.
→Git może tworzyć tzw. gałęzie(branch) jest to odbicie od głównej ścieżki rozwoju aby przetesować inną funkcje nie naruszając głównej struktury programu. →Jednak skąd Git wie na której gałęźi jesteśmy? Utrzymuje on znacznik HEAD nad branchem, w którym pracujemy.
→Same gałęzie można łączyć ze sobą. Git merge szuka commita bazowego spośród końcówek gałęźi, które łączymy. Gdy go znajdzie to utworzy 'commit scalania'. Jednak jeżeli znajdzie konflikt to użytkownik musi zareagować.
→Można podejrzeć historię repozytorium. Wyświetla ona wszystkie zatwierdzone zmiany w odwrotnej kolejności czyli najnowsze na samej górze. Jest wiele możliwości wyświetlenia logu.
. →Istnieją też zdalne repozytoria takie jak GitHub. Zdalne repozytorium to wersja twojego projektu utrzymywana na serwerze dostępnym poprzez Internet lub inną sieć. Możesz mieć ich kilka, z których każde może być tylko do odczytu lub zarówno odczytu jak i zapisu.
→Ale aby wysłać trzeba najpierw dodać połączyć swoje konto np. na GitHub z gitem loklanym. Po stworzeniu zdalnego repozytorium np. GitHub sam kieruje użytkownika do połączenia się z kontem.

⇒Przydatne Komendy Do Git

➺Do tworzenia repozytorium - 'git init'
➺Do dodania lub zaczęcia śledzienia plików - 'git add (nazwa pliku lub '.' aby dodać wszystkie pliki)'
➺Aby usunąć go z jakiegoś stanu - 'git restore (nazwa pliku)'
➺Aby utworzyć nową gałąź (nie zostaniemy na nią przeeniesieni) - 'git branch (nazwa)'
➺Aby przejść na nową gałąź - 'git checkout (nazwa gałęzi)'
➺Aby połączenia gałęzie - 'git merge (nazwa gałęzi)'
➺Do sprawdzenia listy zmian - 'git log'
➺Aby pobrać pliki ze zdalnego repozytorium ale nie zastąpić - 'git fetch'
➺Aby pobrać i odrazu zastąpić - 'git pull'
➺Aby wysłać na zdalne repozytorium - 'git push'
